# Тест-кейсы: Поиск — не сохраняется фильтр по станции метро

## ПОЗИТИВНЫЕ СЦЕНАРИИ

### TC-001: Метро подставляется в фильтры при переходе из вакансии
**Предусловия**: Вакансия с формат работы "Офис", город "Москва", метро "Водный стадион", галочка "Учитывать в поиске станцию метро" включена
**Шаги**:
1. Открыть вакансию /recruiter/vacancy/200
2. Перейти в поиск кандидатов
3. Открыть фильтры поиска

**Ожидаемый результат**:
- Город "Москва" подставлен
- Станция метро "Водный стадион" подставлена
- Поиск выполнен с учётом станции метро

### TC-002: Несколько станций метро подставляются в фильтры
**Предусловия**: Вакансия с несколькими станциями метро и включённой галочкой
**Шаги**:
1. Открыть вакансию с несколькими станциями
2. Перейти в поиск кандидатов
3. Открыть фильтры

**Ожидаемый результат**:
- Все станции метро из вакансии подставлены в фильтры
- Поиск учитывает все станции

### TC-003: Метро НЕ подставляется если галочка выключена
**Предусловия**: Вакансия с метро, но галочка "Учитывать в поиске станцию метро" ВЫКЛЮЧЕНА
**Шаги**:
1. Открыть вакансию
2. Перейти в поиск

**Ожидаемый результат**:
- Станция метро НЕ подставлена в фильтры
- Поиск выполняется без фильтра по метро

## НЕГАТИВНЫЕ СЦЕНАРИИ

### TC-101: Вакансия без метро
**Предусловия**: Вакансия без указания станции метро
**Шаги**:
1. Открыть вакансию без метро
2. Перейти в поиск

**Ожидаемый результат**:
- Фильтр метро пустой
- Поиск работает без фильтра по метро

### TC-102: Формат работы "Удалёнка" — метро не релевантно
**Предусловия**: Вакансия с формат работы "Удалёнка", метро указано
**Шаги**:
1. Открыть вакансию
2. Перейти в поиск

**Ожидаемый результат**:
- Метро не подставляется (удалёнка — метро не имеет смысла для поиска)
