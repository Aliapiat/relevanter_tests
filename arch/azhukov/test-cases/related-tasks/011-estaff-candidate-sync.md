# Тест-кейсы: Синхронизация статусов кандидатов с E-Staff (TASKNEIROKLYUCH-435)

## ПОЗИТИВНЫЕ СЦЕНАРИИ

### TC-001: Создание рассылки с привязанной E-Staff вакансией
**Предусловия**: Вакансия с source-link на E-Staff
**Шаги**: Создать рассылку
**Ожидаемый результат**: Рассылка создана, E-Staff вакансия определена

### TC-002: Создание рассылки БЕЗ E-Staff вакансии → запрос названия
**Предусловия**: Вакансия без source-link на E-Staff
**Шаги**: Создать рассылку
**Ожидаемый результат**: Информационное сообщение + поле ввода названия вакансии

### TC-003: Первый контакт → статус "Первый контакт" в E-Staff
**Предусловия**: Кандидат в рассылке, E-Staff вакансия привязана
**Шаги**: Отправить первое сообщение
**Ожидаемый результат**: В E-Staff: кандидат привязан к вакансии, статус = event_type_1, источник = 73

### TC-004: HR-интервью → статус "Телефонное интервью" в E-Staff
**Предусловия**: Кандидат записался на HR-интервью
**Шаги**: Подтвердить запись
**Ожидаемый результат**: В E-Staff: статус = interview, дата/время = из записи

### TC-005: AI-скрининг → отчёт в приложения E-Staff
**Предусловия**: Кандидат прошёл AI-скрининг
**Шаги**: Скрининг завершён
**Ожидаемый результат**: В E-Staff: отчёт добавлен в "Приложения", статус не изменён

## НЕГАТИВНЫЕ СЦЕНАРИИ

### TC-101: E-Staff API недоступен → рассылка продолжает работать
**Предусловия**: HR-Proxy не отвечает
**Шаги**: Отправить сообщение
**Ожидаемый результат**: Сообщение отправлено, ошибка E-Staff залогирована, не блокирует
