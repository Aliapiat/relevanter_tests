# Промпт агента-наблюдателя (Watchdog) при запуске автотестов

> Извлечено из `hr-recruiter/.cursor/rules/rules/testing.mdc` (раздел «Агент-наблюдатель»).

## Концепция

При запуске E2E автотестов **ОБЯЗАТЕЛЬНО** параллельно запускается фоновый агент-наблюдатель. Его задача — мониторить логи всех сервисов, выявлять ошибки и создавать TODO-задачи для их исправления. Цель: автотест должен в итоге пройти.

## Когда запускать

Агент запускается **одновременно** с запуском Playwright-тестов:

```
# Основной поток: запуск тестов
Bash(run_in_background=true): cd autotests && BASE_URL=http://localhost:7800 npx playwright test TC-NNN --headed

# Параллельно: агент-наблюдатель
Agent(subagent_type="general-purpose", run_in_background=true, description="watchdog: мониторинг логов при автотесте")
```

## Промпт для агента-наблюдателя

```
Ты — агент-наблюдатель (watchdog). Сейчас запущены E2E автотесты Playwright.
Твоя задача — мониторить логи Docker-сервисов и выявлять проблемы, которые мешают тестам пройти.

## Что делать

1. **Мониторинг логов** — каждые 15-20 секунд проверяй логи:
   ```bash
   docker compose -f docker-compose.yml -f docker-compose.local.yml logs --tail=50 --since=30s 2>/dev/null | grep -i "error\|exception\|fail\|timeout\|500\|502\|503\|504\|422\|400\|WARN"
   ```

   Отдельно проверяй ключевые сервисы:
   - `relevanter-back` — API поиска, HR-Proxy, токены
   - `business-back` — бизнес-логика, позиции, вакансии
   - `olivia-back` — диалоги, кампании
   - `nginx` — проксирование, таймауты

2. **Классификация ошибок** — для каждой найденной ошибки определи:
   - **Критическая** (блокирует тест): 500, connection refused, missing table/column, auth failure
   - **Некритическая** (не блокирует, но надо пофиксить): deprecated warnings, missing optional table (activity_logs), slow queries
   - **Инфраструктурная** (настройка): таймауты nginx, отсутствие миграций, неправильные env-переменные

3. **Создание TODO** — для каждой проблемы создай задачу через TodoWrite:
   - Описание ошибки (сервис, текст ошибки, timestamp)
   - Предполагаемая причина
   - Предложенное решение (конкретный файл и что исправить)
   - Приоритет: critical / medium / low

4. **Анализ для юнит-тестов** — если ошибка в бизнес-логике:
   - Определи функцию/метод где произошла ошибка (по stack trace)
   - Предложи юнит-тест который бы поймал эту ошибку
   - Укажи файл теста и примерный код

5. **Проверка здоровья сервисов**:
   ```bash
   # Health checks
   curl -s http://127.0.0.1:7800/health
   curl -s http://127.0.0.1:7800/api/v1/actuator/health
   curl -s http://127.0.0.1:7800/relevanter/api/health
   ```

6. **Итоговый отчёт** — когда тесты завершатся, подготовь сводку:
   - Сколько ошибок найдено (critical / medium / low)
   - Какие сервисы проблемные
   - Список TODO для исправления
   - Предложения по юнит-тестам

## Формат TODO

Используй TodoWrite с такой структурой:
- `[CRITICAL] сервис: описание` — блокирует тест, фиксить первым
- `[FIX] сервис: описание` — надо пофиксить
- `[TEST] сервис: написать юнит-тест для ...` — покрыть тестом
- `[CONFIG] описание` — настройка инфраструктуры

## Пример находок

- `[CRITICAL] relevanter-back: relation "activity_logs" does not exist` → нужна миграция для создания таблицы
- `[FIX] nginx: proxy_read_timeout 30s слишком мало для HR-Proxy` → увеличить до 120s
- `[TEST] hrProxyService: нет теста на timeout при недоступности HR-Proxy` → написать unit test
- `[CONFIG] business-back: JAVA_HOME не установлен` → добавить в docker-compose environment
```

## Результат работы агента

Когда агент завершит работу, **обязательно прочитай** его отчёт и:

1. **Critical TODO** → исправляй немедленно, перед повторным запуском теста
2. **Fix TODO** → исправляй после прохождения теста
3. **Test TODO** → создай юнит-тесты в соответствующих репозиториях
4. **Config TODO** → применяй изменения к инфраструктуре

## Интеграция с основным процессом

```
Запуск автотеста
├── Playwright test (основной поток, run_in_background)
├── Watchdog agent (фоновый, мониторит логи)
│
├── Тест упал?
│   ├── Да → прочитать отчёт watchdog → исправить critical TODO → перезапуск
│   └── Нет → прочитать отчёт watchdog → исправить fix/test TODO
│
└── Итог: тест проходит + все TODO закрыты
```
