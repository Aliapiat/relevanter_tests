## Планируемые тест-кейсы

### TC-002: Soft validation — предупреждения в реальном времени
**Группа**: group-001-vacancy-edit
**Приоритет**: High

**Шаги**:
1. Открыть форму редактирования вакансии (через TC-001)
2. Ввести title > 100 символов → проверить amber-предупреждение
3. Очистить описание → проверить предупреждение "< 200 символов"
4. Добавить > 1 специализацию → проверить предупреждение
5. Исправить данные → проверить что предупреждения исчезли

**Ожидаемый результат**: Предупреждения появляются/исчезают динамически

### TC-003: Сохранение вакансии
**Группа**: group-001-vacancy-edit
**Приоритет**: High

**Шаги**:
1. Открыть форму редактирования (через TC-001)
2. Изменить название вакансии (добавить " test")
3. Нажать "Сохранить"
4. Дождаться toast "Сохранено"
5. Перезагрузить форму → проверить что название обновилось
6. Вернуть исходное название → Сохранить

**Ожидаемый результат**: Данные сохраняются и доступны после перезагрузки

### TC-004: Экспорт/обновление вакансии на HH.ru
**Группа**: group-002-hh-export
**Приоритет**: Medium

**Шаги**:
1. Открыть форму вакансии, привязанной к HH (hhVacancyId != null)
2. Нажать "Обновить на HH.ru"
3. Подтвердить диалог
4. Проверить результат (toast: успех или ошибка)

**Ожидаемый результат**: Запрос отправлен, отображается результат

### TC-005: Ошибки валидации при экспорте на HH.ru
**Группа**: group-002-hh-export
**Приоритет**: Medium

**Шаги**:
1. Открыть форму вакансии с невалидными данными для HH
2. Нажать "Обновить на HH.ru"
3. Подтвердить диалог
4. Проверить отображение ошибок валидации (toast.error)

**Ожидаемый результат**: Список ошибок отображается в toast

### TC-006: Авторизация
**Группа**: group-003-auth
**Приоритет**: High

**Шаги**:
1. Открыть страницу логина
2. Ввести неверные данные → ошибка
3. Ввести верные данные → успешный вход
4. Проверить редирект на /recruiter/reports

**Ожидаемый результат**: Неверные данные отклоняются, верные — пускают
